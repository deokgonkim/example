Your RSA-2048 key pair is generated in your browser — the private key never leaves your device. The signed certificate is installed directly into your browser's certificate store.

⏳
Generating RSA-2048 key pair
Running in your browser via WebCrypto API — private key never leaves this device
⏳
Building Certificate Signing Request (CSR)
Encoding your email as Subject CN and SAN rfc822Name
⏳
Sending CSR to CA for signing
Only the CSR (public key + email) is sent — no private key
⏳
Installing certificate into browser
Assembling PKCS#12 bundle and opening the browser's certificate import dialog

📥 How to import your certificate

  1. A system dialog should have opened automatically. Complete the import wizard.
  2. If no dialog appeared, use the save as file fallback link above, then open chrome://settings/certificates → Your certificates → Import…

💡 Chrome and Edge on Windows/macOS use the OS certificate store. The dialog is the Windows Certificate Import Wizard or macOS Keychain Access.

  1. Firefox should have opened a "Password for Certificate Backup" dialog — enter the password you set (or leave blank).
  2. Click OK. Firefox will confirm the import with a success dialog.
  3. You can verify it in about:preferences#privacy → View Certificates… → Your Certificates.

💡 Firefox handles application/x-pkcs12 natively and imports it directly into its own certificate store.

  1. Safari/macOS should have opened Keychain Access automatically.
  2. Choose the login keychain and click Add. Enter the password if prompted.
  3. The certificate appears under My Certificates in Keychain Access.

💡 macOS Keychain is shared between Safari, Chrome and Edge.
